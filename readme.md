# Инструкция по сборке и запуску
Проект использует `Makefile` для автоматизации настройки окружения и запуска.

*   **Подготовка (venv + зависимости):**
    ```bash
    make
    ```
*   **Запуск обучения и оценки:**

    make run

1. Цель работы
Реализовать алгоритм обучения нейронной сети с нуля, используя исключительно библиотеку NumPy (без применения готовых фреймворков глубокого обучения, таких как PyTorch или TensorFlow).

В рамках данной работы была решена расширенная задача: вместо базового бинарного классификатора на синтетических данных реализован многослойный перцептрон (MLP) для классификации реальных изображений одежды из набора данных Fashion-MNIST (10 классов, размерность признакового пространства — 784).

2. Архитектура решения
Для обеспечения читаемости, модульности и масштабируемости программного кода применена объектно-ориентированная архитектура. Базовым элементом является абстрактный интерфейс Layer, от которого наследуются функциональные слои сети.

Выполняет функцию активации f(x)=max(0,x). Обратный проход реализован вычислительно эффективно с помощью применения векторной булевой маски.

2.4. Интеграция конвейера (MLP)

Класс инкапсулирует упорядоченный список слоев.

Метод forward осуществляет прямой проход данных.

Метод backward реализует алгоритм обратного распространения ошибки (Backpropagation), рекурсивно передавая градиенты от выхода ко входу.

Метод fit реализует стохастический градиентный спуск по мини-батчам (Mini-batch Gradient Descent) с обязательным перемешиванием обучающей выборки перед каждой эпохой.

3. Подготовка данных
Разделение выборки: Использован набор данных Fashion-MNIST. Данные разделены на обучающую (70%) и тестовую (30%) выборки с применением стратификации (stratify=y) для сохранения исходного распределения 10 классов.

Векторизация: Написана собственная функция One-Hot Encoding для преобразования скалярных меток классов в бинарные векторы.

Нормализация: Применена Z-score нормализация (центрирование и масштабирование дисперсии). Для предотвращения утечки данных (Data Leakage) параметры среднего значения и стандартного отклонения вычислялись исключительно по обучающей выборке, после чего применялись к тестовой.

4. Экспериментальное исследование гиперпараметров
Для проведения экспериментов был разработан скрипт автоматизированного тестирования. Ниже представлен анализ результатов на основе полученных графиков функции потерь.

4.1. Влияние скорости обучения (Learning Rate)

Исследуемые значения: 0.001, 0.01, 0.1

Результат: Анализ графика показал, что при η=0.1 (зеленая линия) наблюдается наиболее быстрая сходимость в рамках 50 эпох, приводящая к минимальному значению функции потерь на обучающей выборке. Значение η=0.01 (оранжевая линия) обеспечивает плавную и стабильную сходимость без резких скачков. При η=0.001 (синяя линия) шаг алгоритма оказался слишком мал: ошибка убывает крайне медленно и к 50-й эпохе остается на относительно высоком уровне (~0.4).

4.2. Влияние размера мини-батча (Batch Size)

Исследуемые значения: 16, 64, 256

Результат: График демонстрирует зависимость падения ошибки от эпохи для разных размеров батча. Батч размера 16 (синяя линия) показывает наименьшее значение функции потерь на графике, так как при малом батче алгоритм совершает значительно больше обновлений весов за одну эпоху. Батч 256 (зеленая линия) сходится медленнее (около 0.35 к 50-й эпохе), так как количество шагов оптимизатора за эпоху минимально. Значение 64 (оранжевая линия) выступает оптимальным балансом между скоростью вычисления эпохи и качеством сходимости градиента.

4.3. Влияние инициализации весов

Исследуемые подходы: Нулевая инициализация, нормальная инициализация (He, ~0.01), большие значения (N(0,10)).

Результат: Эксперимент полностью подтвердил теоретические предположения. Инициализация нулями (синяя линия) привела к возникновению полной симметрии весов — функция потерь зафиксировалась на отметке ~2.3 (что равно −ln(0.1) для 10 классов) и модель не обучалась. Инициализация большими весами (зеленая линия) привела к дестабилизации сети: ошибка стартовала с отметки выше 6.0 и на протяжении всего обучения держалась на высоком уровне (~4.0). Корректное падение функции потерь к нулю продемонстрировала исключительно масштабированная инициализация (оранжевая линия).
